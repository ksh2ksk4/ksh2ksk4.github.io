1 概要

Raspberry Pi 4 (Memory: 2GB)にRaspbian Buster Liteをインストールしてヘッドレスで運用。

2 Raspbian Buster Lite(Version: Februrary 2020)

2.1 Install1

  • ここ からイメージファイルをダウンロード

    • ファイルハッシュをチェック

      $ sha256sum 2020-02-13-raspbian-buster-lite.zip
12ae6e17bf95b6ba83beca61e7394e7411b45eba7e6a520f434b0748ea7370e8  2020-02-13-raspbian-buster-lite.zip
  • balenaEtcher でイメージファイルをマイクロSDカードへ書き込む
    • zipファイルを伸長する必要はない
  • マイクロSDカードの /boot に ssh という空のファイルを作成
    • このファイルがあるとSSHDが起動する
  • 作成したマイクロSDカードでRPiを起動

2.2 Settings

ユーザ pi でssh2してログイン。

2.2.1 raspi-config

$ sudo raspi-config nonint do_expand_rootfs
$ sudo raspi-config nonint do_hostname acacia
$ sudo raspi-config nonint do_change_timezone Asia/Tokyo
$ sudo raspi-config nonint do_change_locale ja_JP.UTF-8
$ sudo raspi-config nonint do_wifi_country Japan
$ sudo reboot

2.2.2 apt

$ sudo apt update
$ sudo apt upgrade
$ sudo reboot

2.2.3 DHCP

ネットワーク設定はDHCPを利用するが、IPアドレスを固定するようルータ(YAMAHA NVR510)を設定。

dhcp scope bind 1 192.168.126.10 ethernet dc:a6:32:xx:xx:xx
ip host acacia.home 192.168.126.10

dc:a6:32 は Raspberry Pi Trading Ltd のベンダID。

2.2.4 SSH

2.2.4.1 ssh グループへの追加

念のため以下を実行しておく。

$ sudo usermod -a -G ssh pi
2.2.4.2 パスワードレスでログイン

作業用PCで以下を実行。

$ vi ~/.ssh/config

以下を追記。

Host acacia.home
  HostName acacia.home
  User pi
  IdentityFile ~/.ssh/keys/acacia.home.key

$ ssh-keygen -t ecdsa -b 521 -f ~/.ssh/keys/acacia.home.key -C 'acacia.home'
$ ssh-copy-id -i ~/.ssh/keys/acacia.home.key pi@acacia.home

これでパスワードレスにログインできるはず。

$ ssh acacia.home
2.2.4.3 パスワードログインを禁止
$ sudo vi /etc/ssh/ssh_config

以下を追記。

PasswordAuthentication no
ChallengeResponseAuthentication no
UsePAM no

$ sudo systemctl restart ssh
2.2.4.4 ユーザ pi のパスワードを破棄
$ sudo passwd -d pi

2.2.5 ログインメッセージ

これも鬱陶しいので消しておく。

$ sudo vi /etc/motd

2.2.6 Swap

スワップは利用しない。

$ free -h
$ sudo swapoff --all
$ free -h

$ sudo systemctl status dphys-swapfile
$ sudo systemctl stop dphys-swapfile
$ sudo systemctl disable dphys-swapfile
$ sudo systemctl status dphys-swapfile

2.2.7 Avahi

不要なので無効化。

$ sudo systemctl status avahi-daemon
$ sudo systemctl stop avahi-daemon
$ sudo systemctl disable avahi-daemon
$ sudo systemctl status avahi-daemon

2.2.8 /boot/config.txt

以下を追記。

include extraconfig.txt

$ sudo vi /boot/extraconfig.txt

追加設定を /boot/extraconfig.txt に記述してからリブート。

各設定の詳細は ここ を参照。

2.2.8.1 GPU memory

GPUに割り当てるメモリを最小限(16MB)にしておく。

gpu_mem=16
2.2.8.2 Buletooth

利用しないので無効化。

dtoverlay=disable-bt
2.2.8.3 Wifi

利用しないので無効化。

dtoverlay=disable-wifi
2.2.8.4 Overheat protection

SoCの温度が75℃を超えないようにする。

temp_limit=75

2.2.9 vcgencmd

ディスプレイ出力をOffにする。

$ vcgencmd display_power 0

詳細は ここ を参照。

2.2.10 NTP

systemd-timesyncd で時刻同期するようになっているはず。

$ sudo systemctl status systemd-timesyncd

サービスが起動しているか、ブート時にサービスが起動するようになっているか確認。
(NTP service が active になっているか)

$ sudo timedatectl status
               Local time: 日 2020-01-12 04:41:08 JST
           Universal time: 土 2020-01-11 19:41:08 UTC
                 RTC time: n/a
                Time zone: Asia/Tokyo (JST, +0900)
System clock synchronized: yes
              NTP service: active
          RTC in local TZ: no

設定変更。

$ sudo vi /etc/systemd/timesyncd.conf

NTP=ntp.jst.mfeed.ad.jp
FallbackNTP=ntp.nict.jp

$ sudo systemctl restart systemd-timesyncd

2.2.11 fish

$ sudo apt install fish
$ sudo chsh -s /usr/bin/fish pi

以下のコマンドでログインシェルを変更しようとすると、パスワードでの認証を求められる。
(が、パスワードは破棄してあるため認証できない)

$ chsh -s /usr/bin/fish

で、以下のコマンドでログインシェルを変更しようとすると、スーパーユーザ(root)のログインシェルを変更してしまうので注意。

$ sudo chsh -s /usr/bin/fish

3 Backup

マイクロSDカードを作業用PCへ接続して以下を実行。
(デバイスは環境によって異なる)

$ sudo dd if=/dev/disk2 of=./acaciaYYYYMMDD.img bs=64M status=progress
$ bzip2 acaciaYYYYMMDD.img

なお、MacOSでは以下に注意。

  • bs は小文字でないとだめらしい
    • 64M ではなく 64m
  • status はサポートしていない

4 外部HDD共有

ローカルネットワークでHDDを共有する。

4.1 Cryptsetup

  • https://manpages.debian.org/buster/cryptsetup-bin/cryptsetup.8.en.html

4.1.1 LUKS

pi@acacia ~> lsblk
NAME        MAJ:MIN RM  SIZE RO TYPE MOUNTPOINT
sda           8:0    0  3.7T  0 disk
sdb           8:16   0  1.4T  0 disk
mmcblk0     179:0    0  7.5G  0 disk
├─mmcblk0p1 179:1    0  256M  0 part /boot
└─mmcblk0p2 179:2    0  7.2G  0 part /

sda, sdb を暗号化。

$ sudo cryptsetup luksFormat -i 15000 /dev/sda
$ sudo cryptsetup luksFormat -i 15000 /dev/sdb

ディスクをオープン。

$ sudo cryptsetup luksOpen /dev/sda luks-disk-a
$ sudo cryptsetup luksOpen /dev/sdb luks-disk-b

ファイルシステムはexFATにしたいので、exFATをサポートするパッケージをインストール。

$ sudo apt install exfat-utils

ファイルシステムを作成。

$ sudo mkexfatfs /dev/mapper/luks-disk-a
$ sudo mkexfatfs /dev/mapper/luks-disk-b

マウント設定。

  • https://wiki.archlinux.jp/index.php/Fstab
$ sudo vi /etc/fstab

以下を追記。

/dev/mapper/luks-disk-a /mnt/luks-disk-a exfat defaults,noatime,nofail,uid=1000,gid=1000,fmask=0133,dmask=0022 0 0
/dev/mapper/luks-disk-b /mnt/luks-disk-b exfat defaults,noatime,nofail,uid=1000,gid=1000,fmask=0133,dmask=0022 0 0

ファイルのパーミッションを 0644 にするには fmask を 0022 ではなく 0133 にする必要がある。
(ディレクトリと同様に 0777 に対するマスクを設定するようになっているらしい)

マウントポイントを作成してデバイスマッパーをマウント。

$ sudo mkdir -p /mnt/luks-disk-{a,b}
$ sudo mount /mnt/luks-disk-a
$ sudo mount /mnt/luks-disk-b

不要になったらアンマウントしてデバイスマッパーをクローズ。

$ sudo umount /mnt/luks-disk-{a,b}
$ sudo cryptsetup luksClose luks-disk-a
$ sudo cryptsetup luksClose luks-disk-b

4.2 Samba

  • https://wiki.debian.org/SambaServerSimple

パッケージをインストール。

$ sudo apt install samba

$ sudo vi /etc/samba/smb.conf

[LUKS-disk-A]
  comment = LUKS-disk-A
  path = /mnt/luks-disk-a
  browsable = yes
  guest ok = yes
  writable = yes
  force user = pi
  force group = pi
  force create mode = 0644
  force directory mode = 0755

[LUKS-disk-B]
  comment = LUKS-disk-B
  path = /mnt/luks-disk-b
  browsable = yes
  guest ok = yes
  writable = yes
  force user = pi
  force group = pi
  force create mode = 0644
  force directory mode = 0755

サービスを起動する。

$ sudo systemctl restart smbd
$ sudo systemctl status smbd

4.3 HDDのスピンダウン

  • https://www.htpcguides.com/spin-down-and-manage-hard-drive-power-on-raspberry-pi/

hdparm パッケージをインストール。

$ sudo apt install hdparm

10分でスピンダウンするように設定。

$ sudo hdparm -S 120 /dev/sda
$ sudo hdparm -S 120 /dev/sdb

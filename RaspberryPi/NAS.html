1 概要

Raspberry Pi 4 (Memory: 2GB)に openmediavault をインストールして構築する。

2 Raspbian

Raspbian Buster Lite(Version: September 2019)をインストールしてヘッドレスで利用する。

2.0.1 Install

  • ここ からイメージファイルをダウンロード

    • ファイルハッシュをチェック

      $ sha256sum 2019-09-26-raspbian-buster-lite.zip
a50237c2f718bd8d806b96df5b9d2174ce8b789eda1f03434ed2213bbca6c6ff  2019-09-26-raspbian-buster-lite.zip
  • balenaEtcher でイメージファイルをマイクロSDカードへ書き込む
    • zipファイルを伸長する必要はない
  • マイクロSDカードの /boot に ssh という空のファイルを作成
    • このファイルがあるとSSHDが起動する
  • 作成したマイクロSDカードでRPiを起動

2.0.2 Settings

ユーザ pi でssh1してログインする。

2.0.2.1 raspi-config
$ sudo raspi-config nonint do_expand_rootfs
$ sudo raspi-config nonint do_hostname acacia
$ sudo raspi-config nonint do_change_timezone Asia/Tokyo
$ sudo raspi-config nonint do_change_locale ja_JP.UTF-8
$ sudo raspi-config nonint do_wifi_country Japan
$ sudo reboot
2.0.2.2 apt
$ sudo apt update
$ sudo apt upgrade
$ sudo reboot
2.0.2.3 DHCP

ネットワーク設定はDHCPを利用するが、IPアドレスを固定するようルータ(YAMAHA NVR510)を設定する。

dhcp scope bind 1 192.168.126.10 ethernet dc:a6:32:xx:xx:xx
ip host acacia.home 192.168.126.10

dc:a6:32 は Raspberry Pi Trading Ltd のベンダID。

2.0.2.4 SSH
2.0.2.4.1 ssh グループへの追加

OMVをインストールするとユーザ pi でログインできなくなる。
(インストール後はOMVがSSHを管理するようになり、 ssh グループに属していないユーザのログインを許可していない模様)

OMVインストール後もユーザ pi でログインできるよう、予め以下を実行しておく。
念のため以下を実行しておく。

$ sudo usermod -a -G ssh pi

これを実行せずOMVをインストールした場合、OMVの管理ページでユーザ pi を ssh グループに追加する。

2.0.2.4.2 パスワードレスでログイン

作業用PCで以下を実行する。

$ vi ~/.ssh/config

以下を追記。

Host acacia.home
  HostName acacia.home
  User pi
  IdentityFile ~/.ssh/keys/acacia.home.key

$ ssh-keygen -t ecdsa -b 521 -f ~/.ssh/keys/acacia.home.key -C 'acacia.home'
$ ssh-copy-id -i ~/.ssh/keys/acacia.home.key pi@acacia.home

これでパスワードレスにログインできるはず。

$ ssh acacia.home
2.0.2.4.3 パスワードログインを不許可
$ sudo vi /etc/ssh/ssh_config
$ sudo systemctl restart ssh

ssh_config の変更内容はBookmarksを参照。

2.0.2.8 /boot/config.txt

以下を追記。

include extraconfig.txt

$ sudo touch /boot/extraconfig.txt

追加設定は /boot/extraconfig.txt に記述する。
設定変更後はリブート。

各設定の詳細は ここ を参照。

2.0.2.8.1 GPU memory

GPUに割り当てるメモリを最小限(16MB)にしておく。

gpu_mem=16
2.0.2.8.2 Buletooth

利用しないので無効化。

dtoverlay=disable-bt
2.0.2.8.3 Wifi

利用しないので無効化。

dtoverlay=disable-wifi
2.0.2.8.4 Overheat protection

SoCの温度が75℃を超えないようにする。

temp_limit=75
2.0.2.9 vcgencmd

ディスプレイ出力をOffにする。

$ vcgencmd display_power 0

詳細は ここ を参照。

2.0.2.10 NTP

別途対応。
(OMVインストール後、OMV管理下でChronyが稼動するため)
systemd-timesyncd で時刻同期するようになっているはず。

$ sudo systemctl status systemd-timesyncd

サービスが起動しているか、ブート時にサービスが起動するようになっているか確認する。

$ sudo timedatectl status
               Local time: 日 2020-01-12 04:41:08 JST
           Universal time: 土 2020-01-11 19:41:08 UTC
                 RTC time: n/a
                Time zone: Asia/Tokyo (JST, +0900)
System clock synchronized: yes
              NTP service: active
          RTC in local TZ: no

NTP service が active になっているか確認する。

設定変更。

$ sudo vi /etc/systemd/timesyncd.conf

NTP=ntp.jst.mfeed.ad.jp
FallbackNTP=ntp.nict.jp

$ sudo systemctl restart systemd-timesyncd
2.0.2.11 fish
$ sudo apt install fish
$ sudo chsh -s /usr/bin/fish pi

以下のコマンドでログインシェルを変更しようとすると、パスワードでの認証を求められる。
(が、パスワードは破棄してあるため認証できない)

$ chsh -s /usr/bin/fish

で、以下のコマンドでログインシェルを変更しようとすると、スーパーユーザ(root)のログインシェルを変更してしまうので注意。

$ sudo chsh -s /usr/bin/fish

マイクロSDカードを作業用PCへ接続して以下を実行する。
(デバイスは環境によって異なる)

$ sudo dd if=/dev/disk2 of=./acaciaYYYYMMDD.img bs=64M status=progress
$ bzip2 acaciaYYYYMMDD.img

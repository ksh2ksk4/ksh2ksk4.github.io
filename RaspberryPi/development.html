1 概要

Raspberry Pi 4 (Memory: 4GB)に Arch Linux をインストールして構築する。

2 Arch Linux

2.1 Install

Arch Linuxのディスクを作成する。
acacia(NASサーバ)で作業する。

2.1.1 ダウンロード

$ curl -LO 'http://os.archlinuxarm.org/os/ArchLinuxARM-rpi-4-latest.tar.gz'
$ curl -LO 'http://os.archlinuxarm.org/os/ArchLinuxARM-rpi-4-latest.tar.gz.md5'
$ md5sum -c ArchLinuxARM-rpi-4-latest.tar.gz.md5

2.1.2 パーティショニング

$ lsblk
NAME          MAJ:MIN RM  SIZE RO TYPE MOUNTPOINT
sda             8:0    1 58.9G  0 disk
└─sda1        8:1    1 58.9G  0 part
mmcblk0       179:0    0  7.5G  0 disk
├─mmcblk0p1 179:1    0  256M  0 part /boot
└─mmcblk0p2 179:2    0  7.2G  0 part /

/dev/sda が作業対象のマイクロSDカード。

$ sudo parted /dev/sda --script -- mklabel msdos
$ sudo parted /dev/sda --script -- mkpart primary fat32 1 128
$ sudo parted /dev/sda --script -- mkpart primary ext4 128 100%
$ sudo parted /dev/sda --script -- set 1 boot on

確認。

$ sudo parted /dev/sda --script print
Model: TS-RDF5 SD Transcend (scsi)
Disk /dev/sda: 63.2GB
Sector size (logical/physical): 512B/512B
Partition Table: msdos
Disk Flags:

Number  Start   End     Size    Type     File system  Flags
 1      1049kB  128MB   127MB   primary               boot, lba
 2      128MB   63.2GB  63.1GB  primary

フォーマット。

$ sudo mkfs.vfat -F32 /dev/sda1
$ sudo mkfs.ext4 -F /dev/sda2

2.1.3 ファイルコピー

$ sudo mkdir -p /mnt/arch/{boot,root}
$ sudo mount /dev/sda1 /mnt/arch/boot
$ sudo mount /dev/sda2 /mnt/arch/root

$ sudo tar -xf ArchLinuxARM-rpi-4-latest.tar.gz -C /mnt/arch/root
$ sudo mv /mnt/arch/root/boot/* /mnt/arch/boot
$ sudo umount /mnt/arch/boot /mnt/arch/root

以上でArch Linuxのディスクは完成したので、このディスクでRPiを起動する。

2.2.1 Update

$ su -

2.2.2 DHCP

ネットワーク設定はDHCPを利用するが、IPアドレスを固定するようルータ(YAMAHA NVR510)を設定する。

dhcp scope bind 1 192.168.126.11 ethernet dc:a6:32:xx:xx:xx
ip host cypress.home 192.168.126.11

dc:a6:32 は Raspberry Pi Trading Ltd のベンダID。

が、RPiをリブートしても指定したIPアドレスにならない。
別のIPアドレスがリースされるので、MACアドレスがルータへ届いていないっぽい。

# systemctl status dhcpcd
* dhcpcd.service - dhcpcd on all interfaces
     Loaded: loaded (/usr/lib/systemd/system/dhcpcd.service; disabled; vendor preset: disabled)
     Active: inactive (dead)

dhcpcdが動いていない・・・。
(でも、IPアドレスは取得できている)

どうやら systemd-networkd が諸々やっているらしい。

# networkctl list
IDX LINK  TYPE     OPERATIONAL SETUP
  1 lo    loopback carrier     unmanaged
  2 eth0  ether    routable    configured
  3 wlan0 wlan     off         unmanaged

3 links listed.

# cat /etc/systemd/network/eth0.network
[Match]
Name=eth0

[Network]
DHCP=yes
DNSSEC=no

以下を追記。

[DHCP]
ClientIdentifier=mac

# systemctl restart systemd-networkd

これで指定したIPアドレスがリースされる。

2.2.6 Shell

fish に変更する。

# pacman -S fish
# chsh -s /usr/bin/fish

2.2.7 User

2.2.7.1 デフォルトユーザ(alarm)とは別のユーザを追加
# useradd -m -s /usr/bin/fish kishi

一旦、テンポラリのパスワードを設定する。

# passwd kishi

作業用PCからログインできるか確認する。

$ ssh kishi@cypress.home
2.2.7.2 sudoを使えるようにする

パッケージをインストールする。

# pacman -S sudo

設定。

# visudo

以下を追記。

kishi ALL=(ALL) ALL
Defaults:kishi !authenticate

sudoできるか確認する。

$ sudo su -

以降はこのユーザで作業する。

2.2.7.3 root アカウントをロック

sudoできるようになったので、 root アカウントをロックしておく。

$ sudo passwd -l root
$ sudo cat /etc/shadow

暗号化されたパスワードが残ったままなのは嫌なので /etc/shadow ファイルを編集しておく。

$ sudo vi /etc/shadow
2.2.7.4 デフォルトユーザ(alarm)をロック

root アカウントと同様にアカウントをロックする。

$ sudo passwd -l alarm
$ sudo vi /etc/shadow

3 外部HDD共有

ローカルネットワークでHDDを共有する。
このHDDは予め VeraCrypt で暗号化してある。

3.1 Cryptsetup

VeraCryptで暗号化したディスクはCryptsetupでマウントできそう。

$ su -
# lsblk
NAME        MAJ:MIN RM  SIZE RO TYPE MOUNTPOINT
sda           8:0    0  3.7T  0 disk
sdb           8:16   0  3.7T  0 disk
mmcblk0     179:0    0 58.9G  0 disk
|-mmcblk0p1 179:1    0  121M  0 part /boot
`-mmcblk0p2 179:2    0 58.8G  0 part /

sda, sdb が暗号化したディスク。

# cryptsetup open /dev/sda vc-disk-a --type tcrypt --veracrypt
Enter passphrase for /dev/sda:
# cryptsetup open /dev/sdb vc-disk-b --type tcrypt --veracrypt
Enter passphrase for /dev/sdb:

VeraCryptでディスクを暗号化したときに設定したパスフレーズを入力する。

# cryptsetup status vc-disk-a
/dev/mapper/vc-disk-a is active.
  type:    TCRYPT
  cipher:  aes-xts-plain64
  keysize: 512 bits
  key location: dm-crypt
  device:  /dev/sda
  sector size:  512
  offset:  256 sectors
  size:    7814036656 sectors
  skipped: 256 sectors
  mode:    read/write
# cryptsetup status vc-disk-b
/dev/mapper/vc-disk-b is active.
  type:    TCRYPT
  cipher:  aes-xts-plain64
  keysize: 512 bits
  key location: dm-crypt
  device:  /dev/sdb
  sector size:  512
  offset:  256 sectors
  size:    7814036656 sectors
  skipped: 256 sectors
  mode:    read/write
# lsblk
NAME        MAJ:MIN RM  SIZE RO TYPE  MOUNTPOINT
sda           8:0    0  3.7T  0 disk
`-vc-disk-a 253:0    0  3.7T  0 crypt
sdb           8:16   0  3.7T  0 disk
`-vc-disk-b 253:1    0  3.7T  0 crypt
mmcblk0     179:0    0 58.9G  0 disk
|-mmcblk0p1 179:1    0  121M  0 part  /boot
`-mmcblk0p2 179:2    0 58.8G  0 part  /
# ls -l /dev/mapper/
total 0
crw------- 1 root root 10, 236 Dec 17 12:44 control
lrwxrwxrwx 1 root root       7 Jan 26 08:14 vc-disk-a -> ../dm-0
lrwxrwxrwx 1 root root       7 Jan 26 08:15 vc-disk-b -> ../dm-1
# ls -l /dev/dm-*
brw-rw---- 1 root disk 253, 0 Jan 26 08:14 /dev/dm-0
brw-rw---- 1 root disk 253, 1 Jan 26 08:15 /dev/dm-1

これでデバイスマッパー(/dev/mapper/vc-disk-a, /dev/mapper/vc-disk-b)を通して暗号化ディスクにアクセスできる。

アクセスの必要がなくなったら、以下のコマンドでデバイスマッパーを閉じる。

# cryptsetup close vc-disk-a
# cryptsetup close vc-disk-b

3.2 mount

ディスクのファイルシステムはexFATなので、exFATをサポートするパッケージをインストールする。

# pacman -S exfat-utils

マウントポイントを作成してデバイスマッパーをマウントする。

# mkdir /mnt/vc-disk-{a,b}
# mount /dev/mapper/vc-disk-a /mnt/vc-disk-a
# mount /dev/mapper/vc-disk-b /mnt/vc-disk-b

不要になったらアンマウントする。

# umount /mnt/vc-disk-{a,b}

3.3 Samba

パッケージをインストールする。

# pacman -S samba

インストールしても設定ファイルが存在しないため、 これ を元に設定する。

# vi /etc/samba/smb.conf

[global]
  log file = /var/log/samba/%m.log
  security = user
  map to guest = bad user

[VC-disk-A]
  comment = Share folder
  path = /mnt/vc-disk-a
  browsable = yes
  writable = yes
  #read only = yes
  guest ok = yes

[VC-disk-B]
  comment = Share folder
  path = /mnt/vc-disk-b
  browsable = yes
  writable = yes
  #read only = yes
  guest ok = yes

サービスを起動する。

# systemctl start smb
# systemctl status smb

1 概要

Raspberry Pi 4 (Memory: 4GB)に Arch Linux をインストールして構築。

2 Arch Linux

2.1 Install

  • https://wiki.archlinux.jp/index.php/%E4%B8%80%E8%88%AC%E7%9A%84%E3%81%AA%E6%8E%A8%E5%A5%A8%E4%BA%8B%E9%A0%85

Arch Linuxのディスクを作成(acacia で作業)。

2.1.1 ダウンロード

$ curl -LO 'http://os.archlinuxarm.org/os/ArchLinuxARM-rpi-4-latest.tar.gz'
$ curl -LO 'http://os.archlinuxarm.org/os/ArchLinuxARM-rpi-4-latest.tar.gz.md5'
$ md5sum -c ArchLinuxARM-rpi-4-latest.tar.gz.md5

2.1.2 パーティショニング

$ lsblk
NAME          MAJ:MIN RM  SIZE RO TYPE MOUNTPOINT
sda             8:0    1 58.9G  0 disk
└─sda1        8:1    1 58.9G  0 part
mmcblk0       179:0    0  7.5G  0 disk
├─mmcblk0p1 179:1    0  256M  0 part /boot
└─mmcblk0p2 179:2    0  7.2G  0 part /

/dev/sda が作業対象のマイクロSDカード。

$ sudo parted /dev/sda --script -- mklabel msdos
$ sudo parted /dev/sda --script -- mkpart primary fat32 1 128
$ sudo parted /dev/sda --script -- mkpart primary ext4 128 100%
$ sudo parted /dev/sda --script -- set 1 boot on

確認。

$ sudo parted /dev/sda --script print
Model: TS-RDF5 SD Transcend (scsi)
Disk /dev/sda: 63.2GB
Sector size (logical/physical): 512B/512B
Partition Table: msdos
Disk Flags:

Number  Start   End     Size    Type     File system  Flags
 1      1049kB  128MB   127MB   primary               boot, lba
 2      128MB   63.2GB  63.1GB  primary

フォーマット。

$ sudo mkfs.vfat -F32 /dev/sda1
$ sudo mkfs.ext4 -F /dev/sda2

2.1.3 ファイルコピー

$ sudo mkdir -p /mnt/arch/{boot,root}
$ sudo mount /dev/sda1 /mnt/arch/boot
$ sudo mount /dev/sda2 /mnt/arch/root

$ sudo tar -xf ArchLinuxARM-rpi-4-latest.tar.gz -C /mnt/arch/root
$ sudo mv /mnt/arch/root/boot/* /mnt/arch/boot
$ sudo umount /mnt/arch/boot /mnt/arch/root

以上でArch Linuxのディスクは完成。
このディスクでRPiを起動。

2.2.2 DHCP

ネットワーク設定はDHCPを利用するが、IPアドレスを固定するようルータ(YAMAHA NVR510)を設定。

dhcp scope bind 1 192.168.126.11 ethernet dc:a6:32:xx:xx:xx
ip host cypress.home 192.168.126.11

dc:a6:32 は Raspberry Pi Trading Ltd のベンダID。

が、RPiをリブートしても指定したIPアドレスにならない。
別のIPアドレスがリースされるので、MACアドレスがルータへ届いていないっぽい。

# systemctl status dhcpcd
* dhcpcd.service - dhcpcd on all interfaces
     Loaded: loaded (/usr/lib/systemd/system/dhcpcd.service; disabled; vendor preset: disabled)
     Active: inactive (dead)

dhcpcdが動いていない・・・。
(でも、IPアドレスは取得できている)

どうやら systemd-networkd が諸々やっているらしい。

  • https://manpages.debian.org/buster/systemd/systemd.network.5.en.html
    • Arch Linux版が見つからない・・・
# networkctl list
IDX LINK  TYPE     OPERATIONAL SETUP
  1 lo    loopback carrier     unmanaged
  2 eth0  ether    routable    configured
  3 wlan0 wlan     off         unmanaged

3 links listed.

# cat /etc/systemd/network/eth0.network
[Match]
Name=eth0

[Network]
DHCP=yes
DNSSEC=no

以下を追記。

[DHCP]
ClientIdentifier=mac

# systemctl restart systemd-networkd

これで指定したIPアドレスがリースされる。

2.2.7 User

2.2.7.1 デフォルトユーザ(alarm)とは別のユーザを追加
# useradd -m -s /usr/bin/fish kishi

一旦、テンポラリのパスワードを設定。

# passwd kishi

作業用PCからログインできるか確認。

$ ssh kishi@cypress.home
2.2.7.2 sudoを使えるようにする
  • https://wiki.archlinux.jp/index.php/Sudo

パッケージをインストール。

# pacman -S sudo

設定。

# visudo

以下を追記。

kishi ALL=(ALL) ALL
Defaults:kishi !authenticate

sudoできるか確認。

$ sudo su -

以降はこのユーザで作業を実行。

2.2.7.3 root アカウントをロック

sudoできるようになったので、 root アカウントをロックしておく。

$ sudo passwd -l root
$ sudo cat /etc/shadow

暗号化されたパスワードが残ったままなのは嫌なので /etc/shadow ファイルを編集しておく。

$ sudo vi /etc/shadow
2.2.7.4 デフォルトユーザ(alarm)をロック

root アカウントと同様にアカウントをロック。

$ sudo passwd -l alarm
$ sudo vi /etc/shadow

2.2.8 SSH

2.2.8.1 パスワードレスにログインできるようにする。

作業用PCで以下を実行。

$ ssh-keygen -t ecdsa -b 521 -f ~/.ssh/keys/cypress.home.key -C 'cypress.home'
$ ssh-copy-id -i ~/.ssh/keys/cypress.home.key kishi@cypress.home

$ vi ~/.ssh/config

以下を追記。

Host cypress.home
  HostName cypress.home
  User kishi
  IdentityFile ~/.ssh/keys/cypress.home.key

これでパスワードレスにログインできるはず。

2.2.8.2 追加したユーザのテンポラリパスワードを削除

3 外部HDD共有

ローカルネットワークでHDDを共有。

3.1 Cryptsetup

  • https://wiki.archlinux.jp/index.php/Dm-crypt
  • https://wiki.archlinux.jp/index.php/Dm-crypt/%E3%83%87%E3%83%90%E3%82%A4%E3%82%B9%E3%81%AE%E6%9A%97%E5%8F%B7%E5%8C%96

3.1.1 VeraCrypt

  • https://wiki.archlinux.jp/index.php/TrueCrypt

VeraCryptで暗号化したディスクはCryptsetupでマウントできる。

$ lsblk
NAME        MAJ:MIN RM  SIZE RO TYPE MOUNTPOINT
sda           8:0    0  3.7T  0 disk
sdb           8:16   0  3.7T  0 disk
mmcblk0     179:0    0 58.9G  0 disk
|-mmcblk0p1 179:1    0  121M  0 part /boot
`-mmcblk0p2 179:2    0 58.8G  0 part /

sda, sdb が暗号化したディスク。

ディスクをオープン。

$ sudo cryptsetup open /dev/sda vc-disk-a --type tcrypt --veracrypt
$ sudo cryptsetup open /dev/sdb vc-disk-b --type tcrypt --veracrypt

これでデバイスマッパー(/dev/mapper/vc-disk-a, /dev/mapper/vc-disk-b)を通してデータにアクセスできる。

マウントポイントを作成してデバイスマッパーをマウント。

$ sudo mkdir /mnt/vc-disk-{a,b}
$ sudo mount /dev/mapper/vc-disk-a /mnt/vc-disk-a
$ sudo mount /dev/mapper/vc-disk-b /mnt/vc-disk-b

不要になったらアンマウントしてデバイスマッパーをクローズ。

$ sudo umount /mnt/vc-disk-{a,b}
$ sudo cryptsetup close vc-disk-a
$ sudo cryptsetup close vc-disk-b

3.1.2 LUKS

  • https://gitlab.com/cryptsetup/cryptsetup/-/wikis/FrequentlyAskedQuestions
  • https://wiki.archlinux.jp/index.php/%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB%E3%82%B7%E3%82%B9%E3%83%86%E3%83%A0
  • https://think-t.hatenablog.com/entry/20140112/p1
  • https://jehupc.exblog.jp/19078309/
$ lsblk

sda, sdb を暗号化。

$ sudo cryptsetup luksFormat -i 15000 /dev/sda
$ sudo cryptsetup luksFormat -i 15000 /dev/sdb

ディスクをオープン。

$ sudo cryptsetup luksOpen /dev/sda luks-disk-a
$ sudo cryptsetup luksOpen /dev/sdb luks-disk-b

ファイルシステムはexFATにしたいので、exFATをサポートするパッケージをインストール。

$ sudo pacman -S exfat-utils

ファイルシステムを作成。

$ sudo mkexfatfs /dev/mapper/luks-disk-a
$ sudo mkexfatfs /dev/mapper/luks-disk-b

マウント設定。

$ sudo vi /etc/fstab

以下を追記。

/dev/mapper/luks-disk-a /mnt/luks-disk-a exfat defaults,noatime,nofail,uid=1001,gid=1001,fmask=0133,dmask=0022 0 0
/dev/mapper/luks-disk-b /mnt/luks-disk-b exfat defaults,noatime,nofail,uid=1001,gid=1001,fmask=0133,dmask=0022 0 0k

ファイルのパーミッションを 0644 にするには fmask を 0022 ではなく 0133 にする必要がある。
(ディレクトリと同様に 0777 に対するマスクを設定するようになっているらしい)

マウントポイントを作成してデバイスマッパーをマウント。

$ sudo mkdir -p /mnt/luks-disk-{a,b}
$ sudo mount /mnt/luks-disk-a
$ sudo mount /mnt/luks-disk-b

不要になったらアンマウントしてデバイスマッパーをクローズ。

$ sudo umount /mnt/luks-disk-{a,b}
$ sudo cryptsetup luksClose luks-disk-a
$ sudo cryptsetup luksClose luks-disk-b

3.2 Samba

  • https://wiki.archlinux.jp/index.php/Samba

パッケージをインストール。

$ sudo pacman -S samba

インストールしても設定ファイルが存在しないため、 これ を元に設定。

# vi /etc/samba/smb.conf

[global]
  log file = /var/log/samba/%m.log
  security = user
  map to guest = bad user

[LUKS-disk-A]
  comment = LUKS-disk-A
  path = /mnt/vc-disk-a
  browsable = yes
  guest ok = yes
  writable = yes
  force user = kishi
  force group = kishi
  force create mode = 0644
  force directory mode = 0755

[LUKS-disk-B]
  comment = LUKS-disk-B
  path = /mnt/vc-disk-b
  browsable = yes
  guest ok = yes
  writable = yes
  force user = kishi
  force group = kishi
  force create mode = 0644
  force directory mode = 0755

サービスを起動。

# systemctl start smb
# systemctl status smb

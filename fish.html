4.2 Environment Variables

4.2.1 LANG について

このFAQにある事象が発生1した。

どうやらfish起動前に LANG が定義されているらしく、 

$ set -qg LANG; echo $status
0
$ set |grep LANG
LANG ''

という状態になっている。
(設定値が空のグローバル変数が存在している) 

$ set -qx LANG; echo $status
0
$ set -qU LANG; echo $status
0
$ set -qg LANG; and echo $LANG

$ set -qg LANG; or echo $LANG

ユニバーサル変数のエクスポートもできてはいるが、ユニバーサル変数よりもグローバル変数の方が スコープは優位であるため、設定値を参照できていない模様。 

$ set -e LANG
$ set |grep LANG
LANG ja_JP.UTF-8

グローバル変数を削除すると設定値を参照できる。
そもそも、ユニバーサル変数をエクスポートするのはあまり宜しくなさそうなので、 config.fish で 設定する。 

set -x LANG ja_JP.UTF-8

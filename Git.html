3 特定リポジトリのコミットメッセージに定形文を設定する

prepare-commit-msg を変更する。
リポジトリ毎に変更するのは煩雑なので、変更は一箇所に集約し、各リポジトリからシンボリックリンクを張る。 

#!/usr/bin/env fish

# https://git-scm.com/docs/githooks

set -l argc (count $argv)
set -l message_file $argv[1]

set -l repository_url (git remote -vv |grep '\(fetch\)' |string replace -r '[ \t]+' ' ' |cut -d ' ' -f 2)

string match -r '(^.+/wiki$|^.+ksh2ksk4/memoranda.git$|^.+ksh2ksk4/ksh2ksk4.github.io.git$)' $repository_url

if test $status -eq 0
  # wikiかMemorandaの場合
  echo 'Update documents' |cat - $message_file >"$message_file"_tmp
  mv "$message_file"_tmp $message_file
else
  # wikiかMemoranda以外の場合
  set -l branch (git branch |grep '^* ' |string replace '* ' '')

  string match -r '^feature_' $branch

  if test $status -eq 0
    # フィーチャーブランチの場合
    set -l feature (string replace -r '^feature_' '#' $branch)

    if test $argc -eq 1
      # 通常のコミットの場合
      echo "refs $feature " |cat - $message_file >"$message_file"_tmp
      mv "$message_file"_tmp $message_file
    end
  end
end

$ cd repository
$ ln -sfv ~/.config/fish/functions/git/hooks/prepare-commit-msg .git/hooks/

5 tag

5.1 タグを作成

5.1.1 lightweight tag

$ git tag TAGNAME

5.1.2 annotated tag

$ git tag -a TAGNAME

5.2 タグをリモートへ反映

$ git push origin TAGNAME

5.3 すべてのタグをリモートへ反映

$ git push origin --tags

5.4 タグを削除

$ git tag -d TAGNAME

5.5 リモートからタグを削除

$ git push --delete origin TAGNAME

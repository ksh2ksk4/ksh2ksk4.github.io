以降、Rust and WebAssemblyの4. Tutorialから。

4.1. Setup

Rust toolchain

インストールは済んでいるはず。
Rust 1.30以降が必要。

wasm-pack

.wasm をビルドしたり、テストしたり、公開したりするツール。 

$ curl https://rustwasm.github.io/wasm-pack/installer/init.sh -sSf | sh

cargo-generate

既に存在するリポジトリをテンプレートとしてRustプロジェクトをセットアップするツール。 

$ cargo install cargo-generate

npm

JavaScriptのパッケージマネージャ。
bundlerと開発用サーバのために使用する。
チュートリアルの最後では作成した .wasm をnpmレジストリに公開する。 

$ brew install node
$ npm install npm@latest -g

4.2. Hello, World!

Hello, World! というメッセージを表示するWebページを作成する。

Clone the Project Template

テンプレートをクローンしてプロジェクトを作成。 

$ cargo generate --git https://github.com/rustwasm/wasm-pack-template

プロジェクトの名称は wasm-game-of-life 。

What's Inside

各ファイルの詳細をまとめる。

Build the Project

$ cd wasm-game-of-life
$ wasm-pack build

wasm-pack build でやっていること。

  • Rust 1.30以降がインストールされていることを確認
  • rustup で wasm32-unknown-unknown ターゲットがインストールされていることを確認
  • cargo でRustのコードを .wasm へコンパイル
  • コンパイルされた .wasm を実行するため wasm-bindgen でJavaScript APIを生成

各ファイルの詳細をまとめる。

Putting it into a Web Page

$ npm init wasm-app www

wasm-game-of-life パッケージをWebページで使えるようにするため、create-wasm-appテンプレートを元にJavaScriptプロジェクトを作成する。

各ファイルの詳細をまとめる。

Install the dependencies

$ cd www
$ npm install

開発用サーバとそれに必要なものをインストール。
(RustとWebAssemblyに webpack は必須ではない。便利なのでインストールしている)

Using our Local wasm-game-of-life Package in www

$ cd ../pkg
$ npm link
/usr/local/lib/node_modules/wasm-game-of-life -> /Users/kishi/.own/srcs/Rust/wasm-game-of-life/pkg

このシンボリックリンクにより、ローカルの wasm-game-of-life パッケージが npm でインストールされたパッケージのように見える。 

$ cd ../www
$ npm link wasm-game-of-life
/Users/kishi/.own/srcs/Rust/wasm-game-of-life/www/node_modules/wasm-game-of-life -> /usr/local/lib/node_modules/wasm-game-of-life -> /Users/kishi/.own/srcs/Rust/wasm-game-of-life/pkg

www パッケージが wasm-game-of-life パッケージ(さきほど npm link した wasm-game-of-life パッケージ)を参照するようにしている。 

import * as wasm from "wasm-game-of-life";

wasm.greet();

JavaScriptで wasm-game-of-life パッケージをインポートするように変更。

Serving Locally

別のターミナルで以下を実行する。
(開発用Webサーバが起動し続けるため) 

$ cd wasm-game-of-life/www
$ npm run start

ブラウザで http://localhost:8080/ へアクセス。

wasm-game-of-life ディレクトリで以下のコマンドを実行すると、コードの変更が反映される。 

$ wasm-pack build

Exercises

アラートメッセージをカスタマイズするために wasm-game-of-life/src/lib.rs の greet 関数が name: &str パラメータを取るように変更して、 wasm-game-of-life/www/index.js から greet 関数へ名前を渡す。
wasm-pack build で .wasm ファイルをリビルドしてからブラウザで http://localhost:8080/ を再表示してカスタマイズしたメッセージが表示されることを確認する。

変更前

extern crate cfg_if;
extern crate wasm_bindgen;

mod utils;

use cfg_if::cfg_if;
use wasm_bindgen::prelude::*;

cfg_if! {
    // When the `wee_alloc` feature is enabled, use `wee_alloc` as the global
    // allocator.
    if #[cfg(feature = "wee_alloc")] {
        extern crate wee_alloc;
        #[global_allocator]
        static ALLOC: wee_alloc::WeeAlloc = wee_alloc::WeeAlloc::INIT;
    }
}

#[wasm_bindgen]
extern {
    fn alert(s: &str);
}

#[wasm_bindgen]
pub fn greet() {
    alert("Hello, wasm-game-of-life!");
}

import * as wasm from "wasm-game-of-life";

wasm.greet();

変更後

extern crate cfg_if;
extern crate wasm_bindgen;

mod utils;

use cfg_if::cfg_if;
use wasm_bindgen::prelude::*;

cfg_if! {
    // When the `wee_alloc` feature is enabled, use `wee_alloc` as the global
    // allocator.
    if #[cfg(feature = "wee_alloc")] {
        extern crate wee_alloc;
        #[global_allocator]
        static ALLOC: wee_alloc::WeeAlloc = wee_alloc::WeeAlloc::INIT;
    }
}

#[wasm_bindgen]
extern {
    fn alert(s: &str);
}

#[wasm_bindgen]
pub fn greet(name: &str) {
    alert(&format!("Hello, {}!", name));
}

import * as wasm from "wasm-game-of-life";

wasm.greet("foo");

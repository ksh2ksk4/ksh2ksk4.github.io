2 Data types

2.1 Scalar types

2.1.1 整数型(符号あり)

  Size(bit) Min Max
i8 8 -256 127
i16 16 -32768 32767
i32 32 -2147483648 2147483647
i64 64 -9223372036854775808 9223372036854775807
isize 1      

取り得る値の範囲は以下の通り。

\[ -(2^{n-1}) \leq x \leq 2^{n-1} - 1 \]

i32 が整数型の基準型。

2.1.2 整数型(符号なし)

  Size(bit) Min Max
u8 8 0 255
u16 16 0 65535
u32 32 0 4294967295
u64 64 0 18446744073709551615
usize 1      

取り得る値の範囲は以下の通り。

\[ 0 \leq x \leq 2^n - 1 \]

2.1.3 浮動小数点数型

f32 (単精度浮動小数点数) と f64 (倍精度浮動小数点数) があり、 f64 が浮動小数点数型の基準型。

2.1.4 論理値型

論理値型は bool であり、 true と false が取り得る値。

2.1.5 文字型

文字型は char であり、Unicodeのスカラ値を表す。
文字列とは異なるため、 " ではなく ' でクォートする。

2.1.6 ユニット型

ユニット型は () であり、空のタプル( () )が取り得る値。

3 Variable

3.1 Shadowing

前に定義した変数と同じ名前の新しい変数を定義できる。 

let x = 1;
println!("The value of x is {}", x);
let x = x + 1;
println!("The value of x is {}", x);

The value of x is 1
The value of x is 2

ミュータブルではないので再束縛は不可(コンパイルエラー)。 

let x = 1;
x = 100;

error[E0384]: cannot assign twice to immutable variable `x`
 --> rust-src-KLc4WQ.rs:3:1
  |
2 | let x = 1;
  |     - first assignment to `x`
3 | x = 100;
  | ^^^^^^^ cannot assign twice to immutable variable

型を変更することも可能。 

let spaces = "        ";
let spaces = spaces.len();
println!("The value of spaces is {}", spaces);

The value of spaces is 8

4 Function

  • main はプログラムのエントリポイント
  • snake_case で命名するのが慣例
  • 関数定義では仮引数2の型は宣言しなければならない
  • 戻り値は -> の後に型を指定して宣言しなければならない
fn main() {
    println!("The return value of sub_function is {}", sub_function(10));
}

fn sub_function(x: i32) -> i32 {
    println!("The value of x is {}", x);
    x
}

The value of x is 10
The return value of sub_function is 10
  • 文(statement)
    • 実行後に値を返さない
    • 終端に ; を含む
    • 関数定義は文
  • 式(expression)
    • 実行後に値を返す
    • 終端に ; を含まない
    • ブロック( {} )は式

5 フロー制御

5.1 if

条件式は bool でなければならない。
(bool 以外の値が暗黙的に変換されることはない) 

if true {
    println!("true");
} else {
    println!("false");
}

true

条件が複数存在する場合、 else if を利用できる。
また、 if は文ではなく式であるため、文や式の右辺に置くことができる。 

let result = if true {
    true
} else {
    false
};

println!("The value of result is {}", result);

The value of result is true

7 構造体(struct)

7.1 構造体

// 構造体を定義
#[derive(Debug)]
struct User {
    username: String,
    email: String,
    sign_in_count: u64,
    active: bool
}

// 不変インスタンスを生成
let user1 = User {
    username: String::from("foo"),
    email: String::from("foo@example.com"),
    active: true,
    sign_in_count: 1
};
println!("The value of user1 is {:?}", user1);

// 可変インスタンスを生成
// 一部のフィールドのみ可変にすることはできない。インスタンス全体を可変にする。
let mut user2 = User {
    username: String::from("bar"),
    email: String::from("bar@example.com"),
    sign_in_count: 1,
    active: true
};
user2.active = false;
println!("The value of user2 is {:?}", user2);

let user3 = User {
    username: String::from("baz"),
    email: String::from("baz@example.com"),
    ..user1
};
println!("The value of user3 is {:?}", user3);

// 不変インスタンスを生成する関数
fn build_user(username: String, email: String) -> User {
    User {
        username,
        email,
        sign_in_count: 1,
        active: true
    }
}

let user4 = build_user(String::from("foo"), String::from("foo@example.com"));
println!("The value of user4 is {:?}", user4);

The value of user1 is User { username: "foo", email: "foo@example.com", sign_in_count: 1, active: true }
The value of user2 is User { username: "bar", email: "bar@example.com", sign_in_count: 1, active: false }
The value of user3 is User { username: "baz", email: "baz@example.com", sign_in_count: 1, active: true }
The value of user4 is User { username: "foo", email: "foo@example.com", sign_in_count: 1, active: true }

7.2 タプル構造体

フィールドに名前の不要な構造体。

#[derive(Debug)]

let black = Color(0, 0, 0);
println!("The value of black is {:?}", black);
let origin = Point(-1, -1, -1);

The value of black is Color(0, 0, 0)
The value of origin is Point(-1, -1, -1)

7.7 プログラム例

7.7.1 四角形の面積を算出する

let width = 30;
let height = 50;
println!("The area of the rectangle is {} square pixels.", area(width, height));

fn area(width: u32, height: u32) -> u32 {
    width * height
}

The area of the rectangle is 1500 square pixels.

let rectangle = (30, 50);
println!("The area of the rectangle is {} square pixels.", area(rectangle));

fn area(dimensions: (u32, u32)) -> u32 {
    dimensions.0 * dimensions.1
}

The area of the rectangle is 1500 square pixels.

#[derive(Debug)]
struct Rectangle {
    width: u32,
    height: u32
}

let rectangle = Rectangle{width: 30, height: 50};
println!("rectangle: {:#?}", rectangle);
println!("The area of the rectangle is {} square pixels.", area(&rectangle));

fn area(rectangle: &Rectangle) -> u32 {
    rectangle.width * rectangle.height
}

rectangle: Rectangle {
    width: 30,
    height: 50
}
The area of the rectangle is 1500 square pixels.

#[derive(Debug)]
struct Rectangle {
    width: u32,
    height: u32
}

impl Rectangle {
    fn area(&self) -> u32 {
        self.width * self.height
    }
}

let rectangle = Rectangle{width: 30, height: 50};
println!("rectangle: {:#?}", rectangle);
println!("The area of the rectangle is {} square pixels.", rectangle.area());

rectangle: Rectangle {
    width: 30,
    height: 50
}
The area of the rectangle is 1500 square pixels.

7.7.2 他の四角形を内包するかチェックする

#[derive(Debug)]
struct Rectangle {
    width: u32,
    height: u32
}

impl Rectangle {
    fn area(&self) -> u32 {
        self.width * self.height
    }

    fn can_hold(&self, other: &Rectangle) -> bool {
        self.width > other.width && self.height > other.height
    }
}

let rectangle1 = Rectangle{width: 30, height: 50};
let rectangle2 = Rectangle{width: 10, height: 40};
let rectangle3 = Rectangle{width: 60, height: 45};
println!("rectangle1: {:#?}", rectangle1);
println!("rectangle2: {:#?}", rectangle2);
println!("rectangle3: {:#?}", rectangle3);
println!("Can rectangle1 hold rectangle2? {}", rectangle1.can_hold(&rectangle2));
println!("Can rectangle1 hold rectangle3? {}", rectangle1.can_hold(&rectangle3));

rectangle1: Rectangle {
    width: 30,
    height: 50
}
rectangle2: Rectangle {
    width: 10,
    height: 40
}
rectangle3: Rectangle {
    width: 60,
    height: 45
}
Can rectangle1 hold rectangle2? true
Can rectangle1 hold rectangle3? false

7.7.3 正方形を生成する

#[derive(Debug)]
struct Rectangle {
    width: u32,
    height: u32
}

impl Rectangle {
    fn square(size: u32) -> Rectangle {
        Rectangle {width: size, height: size}
    }
}

let square = Rectangle::square(100);
println!("square: {:#?}", square);

square: Rectangle {
    width: 100,
    height: 100
}

8 Enumとパターンマッチング

Rustの列挙型(enum)は関数型言語における代数的データ型に非常に近い。

8.1 e.g. IPアドレスデータを表現する場合

8.1.1 列挙型と構造体によるIPアドレスデータ

#[derive(Debug)]
enum IpAddrKind {
    V4,
    V6
}

#[derive(Debug)]
struct IpAddr {
    kind: IpAddrKind,
    address: String
}

let home = IpAddr {
    kind: IpAddrKind::V4,
    address: String::from("127.0.0.1")
};
let loopback = IpAddr {
    kind: IpAddrKind::V6,
    address: String::from("::1")
};
println!("home: {:#?}", home);
println!("loopback: {:#?}", loopback);

home: IpAddr {
    kind: V4,
    address: "127.0.0.1"
}
loopback: IpAddr {
    kind: V6,
    address: "::1"
}

8.1.2 列挙型のみによるIPアドレスデータ

列挙型では列挙子に任意のデータを関連付けることができる。 

#[derive(Debug)]
enum IpAddr {
    V4(u8, u8, u8, u8),
    V6(String)
}

let home = IpAddr::V4(127, 0, 0, 1);
let loopback = IpAddr::V6(String::from("::1"));
println!("home: {:#?}", home);
println!("loopback: {:#?}", loopback);

home: V4(
    127,
    0,
    0,
    1
)
loopback: V6(
    "::1"
)

2.3.2 エクスポート実行時にコードブロックの実行確認をしない

(setq org-confirm-babel-evaluate nil)

2.3.3 Header arguments

  • :exports
    • code
      • コード本体をエクスポート。デフォルト
    • results
      • コードの実行結果をエクスポート
    • both
      • コード本体とコードの実行結果をエクスポート
    • none
      • 何もエクスポートしない
  • :results この引数には以下の4つのクラスがある。それぞれ1つのオプションを指定できる。
    • Collection
    • Type
    • Format
    • Handling
      • silent
        • Org モードバッファに結果を追加しない。ミニバッファには表示する。
        • org のソースコードを実行すると、これがデフォルトになっている・・・?
      • replace
        • デフォルト。 Org モードバッファに結果を追加する。以前の結果は削除する。
      • append
        • Org モードバッファに結果を追加する。以前の結果は削除しない。最新の結果が一番下になる。
      • prepend
        • Org モードバッファに結果を追加する。以前の結果は削除しない。最新の結果が一番上になる。

3 コードブロックのコードを実行する

コードブロックにカーソルを置いて C-c C-c で実行できる。
実行結果はコードブロック直下の #+RESULT: に出力される。

  • オプション
    • :exports both
      • コードブロックと実行結果をエクスポートする
    • :exports results
      • 実行結果のみエクスポートする

3.1 Calc

3.1.1 org-babel で使用する言語に追加

(defun dotspacemacs/user-config ()
  (org-babel-do-load-languages
   'org-babel-load-languages '((calc . t))))

3.1.2 e.g.

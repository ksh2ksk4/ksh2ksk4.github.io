1.1 4.1. Setup

1.1.1 The Rust Toolchain

Rust 1.30以降が必要。

1.1.2 wasm-pack

.wasm をビルドしたり、テストしたり、公開したりするツール。

$ curl https://rustwasm.github.io/wasm-pack/installer/init.sh -sSf | sh

1.1.3 cargo-generate

既に存在するリポジトリをテンプレートとしてRustプロジェクトをセットアップするツール。

$ cargo install cargo-generate

1.1.4 npm

JavaScriptのパッケージマネージャ。
bundlerと開発用サーバのために使用する。
チュートリアルの最後では作成した .wasm をnpmレジストリに公開する。

$ brew install node
$ npm install npm@latest -g

1.2 4.2. Hello, World!

Hello, World! というアラートを表示するWebページを作成する。

1.2.1 Clone the Project Template

テンプレートをクローンしてプロジェクトを作成。

$ cargo generate --git https://github.com/rustwasm/wasm-pack-template

プロジェクトの名称は wasm-game-of-life 。

1.2.2 What's Inside

$ cd wasm-game-of-life
$ tree
.
├── Cargo.toml
├── LICENSE_APACHE
├── LICENSE_MIT
├── README.md
├── src
│   ├── lib.rs
│   └── utils.rs
└── tests
    └── web.rs

2 directories, 7 files
  • Cargo.toml
    • wasm-bindgen の依存関係が設定されている
    • .wasm ライブラリをビルドするよう crate-type が設定されている
  • src/lib.rs
    • メインファイル
  • src/utils.rs
    • ユーティリティを記述するファイル

1.2.3 Build the Project

$ wasm-pack build
  • Rust 1.30以降がインストールされていることを確認
  • rustup で wasm32-unknown-unknown ターゲットがインストールされていることを確認
  • cargo でRustのコードを .wasm へコンパイル
  • コンパイルされた .wasm を実行するため wasm-bindgen でJavaScript APIを生成

各ファイルの詳細をまとめる。

1.2.4 Putting it into a Web Page

$ npm init wasm-app www

wasm-game-of-life パッケージをWebページで使えるようにするため、 create-wasm-app テンプレートを元にJavaScriptプロジェクトを作成する。

各ファイルの詳細をまとめる。

1.2.5 Install the dependencies

$ cd www
$ npm install

開発用サーバとそれに必要なものをインストール。
(RustとWebAssemblyに webpack は必須ではない。便利なのでインストールしている)

1.2.6 Using our Local wasm-game-of-life Package in www

下記の2ファイルを変更する。

"dependencies": {
  "wasm-game-of-life": "file:../pkg"
}

上記を追記。

import * as wasm from "wasm-game-of-life";

wasm.greet();

その後、以下を実行する。

$ npm install

1.2.7 Serving Locally

$ npm run start

ブラウザで http://localhost:8080/ へアクセスすると、アラートが表示される。

コードの変更を反映するには、 wasm-game-of-life ディレクトリで以下のコマンドを実行する。

$ wasm-pack build
